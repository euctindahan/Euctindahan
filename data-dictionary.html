Marketplace Database Documentation & Data Dictionary

1. User Entity (/users)

Stores authentication details, roles, and MSEUF verification status.

Field Type Required Description
uidString (PK)YesUnique identifier from Firebase Auth.
emailStringYesMSEUF student/faculty email address.
displayNameStringYesFull name of the user.
roleEnumYesadmin | seller | customer
verificationStatusEnumYesunverified | pending | verified | rejected
isBlockedBooleanNoAdministrative restriction toggle.
gcashNumberStringNoPayment account number.

2. Product Entity (/products)

Inventory items created by verified student sellers.

Field Type Required Description
idString (PK)YesUnique product ID (PROD-XXXX).
sellerIdString (FK)YesUID of the student owner.
nameStringYesThe product title.
priceNumberYesUnit price in PHP.
stockNumberYesQuantity available.
categoryStringYesProduct category (e.g., accessories).
imagesArrayYesList of image URLs.

3. Order Entity (/orders)

Log of all marketplace transactions.

Field Type Required Description
idString (PK)YesThe unique order record ID.
customerIdString (FK)YesBuyer's UID.
sellerIdString (FK)YesSeller's UID.
totalNumberYesTotal transaction amount.
statusEnumYesPREPARING | SHIPPED | DELIVERED | CANCELLED
paymentMethodStringYesGCASH or COD
dateTimestampYesTime of sale.

4. System Configuration (/settings)

Global app variables managed by Administrators.

Field Type Default Description
listingTaxRateNumber0.05Platform fee percentage.
